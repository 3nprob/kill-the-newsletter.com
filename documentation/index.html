1 Overview

Kill the Newsletter! is an application that converts email newsletters into Atom feeds. It spins up a web server and an email server. The web server is the front-end that allows for the creation of new Atom feeds. The email server receives emails and saves them as Atom entries. The feeds are stored as files in the file system.

2 Dependencies

Kill the Newsletter! is written in Go. Use Glide to install the package dependencies:

$ glide install

3 Development

Run a local instance of Kill the Newsletter! with:

$ go run kill-the-newsletter.go

Visit http://localhost:8080. Create feeds and send emails to test them. The following command line sends a test email using msmtp:

$ printf 'Subject: Test\n\nHello world of Kill the Newsletters!' | \

  msmtp --host=localhost --port=2525 --from=publisher@example.com -- \

        <feed-token>@localhost

4 Deployment

Because it depends on the file system, it is not possible to deploy Kill the Newsletter! to Heroku.

While the build step requires a Go compiler, the environment that runs the application does not need one.

Do not enable listing the contents of the feeds directory! The file names contain tokens which are sensitive information.

To deploy a self-hosted instance, the recommended environment is a Docker container. See the accompanying Makefile, Dockerfile and docker-compose.yml.

Configure the application and setup a web server in front of it. This acts as reverse proxy and serves the contents of the feeds folder as static files. The following is an example configuration for nginx:

"/etc/nginx/nginx.conf"

# …

 

server {

  listen 443 ssl;

  server_name www.kill-the-newsletter.com;

 

  location / {

    proxy_pass http://kill-the-newsletter:80;

  }

 

  location /feeds {

    root /var/www/www.kill-the-newsletter.com;

  }

}

 

server {

  listen 80;

  server_name www.kill-the-newsletter.com;

 

  return 301 https://www.kill-the-newsletter.com$request_uri;

}

 

server {

  listen 80;

  listen 443 ssl;

  server_name kill-the-newsletter.com;

 

  return 301 https://www.kill-the-newsletter.com$request_uri;

}

 

# …

5 Configuration

See the full configuration for the main Kill the Newsletter! deployed instance. It is more complex, including a reverse proxy in front of the email server.

Configure Kill the Newsletter! with a file named "./kill-the-newsletter.json" in JSON format. The following configuration keys are available:

Key

 

Default

 

Description

Name

 

Kill the Newsletter!

 

The service name, shown in communications to the user.

Administrator

 

mailto:administrator@example.com

 

The system administrator contact reference.

Web.Server

 

:8080

 

The network address on which the web server listens.

Web.URL

 

http://localhost:8080

 

The URL for the application. Used as base for links. Note the lack of a trailing slash.

Web.URIs.Root

 

/

 

The root URI for the application. Note the trailing slash.

Web.URIs.Feeds

 

/feeds/

 

The URI under which to find the feeds. Note the trailing slash.

Email.Server

 

:2525

 

The network address on which the email server listens.

Email.Host

 

localhost

 

The host for which the application accepts emails.

Feed.Path

 

./feeds/

 

The file system path in which to store the feeds as files.

Feed.Suffix

 

.xml

 

The suffix to use for feeds files.

Feed.URN

 

localhost

 

The URN to use when creating identifiers for feeds and entries.

Feed.SizeLimit

 

1900000

 

The maximum size (in bytes) for a feed. Larger feeds are truncated.

Token.Length

 

20

 

The length of the tokens that identify feeds.

Token.Characters

 

abcdefghijklmnopqrstuvwxyz0123456789

 

The characters that form tokens.

Example:

"./kill-the-newsletter.json"

{

  "administrator": "mailto:kill-the-newsletter@leafac.com",

  "web": {

    "server": ":80",

    "url": "https://www.kill-the-newsletter.com"

  },

  "email": {

    "server": ":25",

    "host": "kill-the-newsletter.com"

  },

  "feed": {

    "urn": "kill-the-newsletter"

  }

}
